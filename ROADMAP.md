# ZINTENT Roadmap (Vision -> Shippable)

This repo already ships a **build-time compiler** + **tree-shaken CSS output**. This roadmap maps your feature spec to what exists today, what's partially implemented, and what's planned next.

## Status Legend
- Implemented (in repo)
- Partial (needs hardening / expansion)
- Planned (not implemented yet)
- External/Tooling (depends on runtime/deployment/tools)

## Core Compiler + Output
- Implemented: Zero-runtime CSS (build-time compiler): `compiler/main-v2.go`
- Implemented: Tree-shake unused styles (extract used classes from HTML)
- Partial: Atomic CSS + deduplication (variables exist; no cross-rule dedupe pass yet)
- Implemented: Size budgets + bundle analysis (`--analyze`, `--budget-kb`)
- External/Tooling: HTTP/3 / brotli dictionary optimization (deployment concern, not framework code)

## Modern CSS Architecture
- Implemented: Container queries (container-first): `core/container-queries.css`
- Implemented: Reduced-motion support: generated by compiler
- Implemented: Cascade layers ordering (`@layer zi-*`): `compiler/main-v2.go`
- Implemented: Style scoping & prefixing support (`--scope` flag): `compiler/main-v2.go`
- Planned: Native nesting + `:has()` patterns (add intents + docs)

## Design Tokens + Theming
- Implemented: Token file: `core/tokens.css`
- Implemented: Theme library: `core/themes.css` (includes `light`)

- Implemented: Multi-theme pack (includes `high-contrast`, `midnight`, `vibrant`, `forest`, `nordic`)
- Planned: Runtime theme switching helpers (JS snippet + CSS-only fallback)

## What "Phase 5 Complete" means in this repo (proposed)
Phase 5 should be considered complete when:
1) `npm run build:phase5` reproduces `dist/phase5-styles.css` deterministically
2) Tokens + themes include everything Phase 5 components use (no missing vars)
3) Accessibility basics are shipped (`sr-only`, focus-visible patterns, skip-link focus)
4) Docs describe how to use the Phase 5 interaction patterns without copy/paste JS

5) Advanced Interaction recipes (View Transitions, Scroll-driven) are demoed in Phase 5 file.
6) High-Contrast theme is verified for WCAG 2.2 AAA compliance.
7) Masonry grid intents support cross-browser responsive columns.
